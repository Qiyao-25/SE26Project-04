# 架构落地说明（演示边界）

本文对齐评审中的「功能闭环原型」定位，明确**当前已落地**与**刻意未做**的部分，避免文档与实现漂移。

## 认证与安全（已收紧）

| 能力 | 状态 |
|------|------|
| 登录用户触发解析 `/papers/{id}/parse` | 需 Bearer |
| 查询/重试任务 | 需登录 |
| 批量写入 `/papers/batch`、`/chunks`、入队 `enqueue-pending` | 需 **admin** |
| Worker claim/finalize | `X-Worker-Token`；prod 禁止空/占位符 |
| 调试全站抓取 | 开关 + **admin** |
| 生产 `/docs` | 默认关闭；`deploy-on-host` / compose 默认 `ENABLE_DOCS=false` |
| 生产 `AUTH_SECRET` / `WORKER_TOKEN` | `validate_production_settings` 拒绝弱口令 |

## 解析执行模型

- **主路径**：FastAPI 进程内 `BackgroundTasks` + `parse_agent_runner`
- **正文提取**：优先 PDF（`pypdf`）；文本过短或失败时回退 **ar5iv/HTML**（stdlib `HTMLParser`）；再失败用摘要
- **可选**：PaperPipeline Worker（同源任务表）；错误码需与 `ParseTask` 对齐，课程阶段以主路径为准
- **PDF 文件**：临时下载解析后删除；不做对象存储。`storage_path` 预留，未实现写入流水线

## 检索与问答

- 文本块检索：词法/规则匹配；`embedding` 列未生成、未查询
- 问答：优先 QA Agent；失败或未配置时 `answer_mode=extractive_fallback`，前端展示「降级摘录」标签
- 多模式辅助阅读：优先 `ReadingModeAgent`（`/api/papers/{id}/assist`）；失败时用 `build_fallback_assist` 启发式模板

## 论文去重

1. 优先按归一化 `arxiv_id` 唯一 upsert  
2. 若 ID 未见但归一化标题碰撞 → **合并到已有行**（不新建）

## 后续升级（非本迭代强制）

1. Redis + Celery / 独立 Worker + Outbox  
2. PDF 持久化与 `storage_path` 回填  
3. Embedding + BM25 + reranker 混合检索  
4. 收敛或删除 Pipeline stub，统一错误码
