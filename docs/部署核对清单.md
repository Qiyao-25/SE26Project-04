# PaperMate 服务器部署核对清单

## A. 打包与上传

- [ ] 已生成前端包 `papermate-frontend-*.tar.gz`
- [ ] 已生成后端包 `papermate-backend-*.tar.gz`
- [ ] 已生成配置包 `papermate-deploy-config-*.tar.gz`（或直接用 `deploy/`）
- [ ] 三包已上传服务器并校验大小非 0（可用 `deploy/upload-packages.ps1`）

## B. 后端

- [ ] Python ≥ 3.11，venv 已用系统 `/usr/bin/python3.11` 创建
- [ ] `pip install -e ".[postgres]"` 或 SQLite 场景 `pip install -e .`
- [ ] `.env` 已填写：`AUTH_SECRET`、`CORS_ORIGINS`、`LLM_API_KEY`、`DATABASE_URL`
- [ ] `PAPERMATE_ENV=prod`
- [ ] `alembic upgrade head` 成功
- [ ] （可选）`import_seed` 导入 `seed.json`
- [ ] systemd `papermate-backend` 已 enable 且 `curl 127.0.0.1:8000/health` 成功

## C. 前端与 Nginx

- [ ] 静态文件位于 Nginx root（如 `/var/www/papermate`）
- [ ] `/api` 反代到 `127.0.0.1:8000`
- [ ] `VITE_USE_MOCK=false` 且 API 前缀为 `/api`（构建期）
- [ ] 浏览器打开域名可加载 SPA，无 Vite 开发端口依赖

## D. 功能冒烟

- [ ] 注册 / 登录
- [ ] 工作台推荐有数据
- [ ] 论文解析 → 智能总结 / 校验标记
- [ ] 问答返回出处（含页码）
- [ ] 设置页订阅同步（需出站访问 arXiv）

## E. 安全

- [ ] `.env` 权限 `600`，未进 Git
- [ ] 生产关闭或限制 `/docs`（`ENABLE_DOCS=false`）
- [ ] 后端仅监听本机，不直接对公网暴露 8000

更新流程见 [服务器运维与更新.md](./服务器运维与更新.md)。
