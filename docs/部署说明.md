# PaperMate 环境安装与服务器部署说明

制定目的：支撑迭代3「项目环境安装部署说明」交付，指导在 Linux 服务器上安装前端静态包与后端 API。

## 1. 架构与安装包

正式交付建议准备 **2～3 个安装包**：

| 包名 | 内容 | 是否必需 |
|------|------|----------|
| `papermate-frontend-*.tar.gz` | Vite 构建产物（`dist/`） | 必需 |
| `papermate-backend-*.tar.gz` | FastAPI 后端源码 + 迁移 | 必需 |
| `papermate-deploy-config-*.tar.gz` | Nginx / systemd / 环境模板 / 本文档 | 推荐 |

本地打包（在仓库根目录）：

```bash
python deploy/pack.py
# 默认产物：dist/packages/
```

**部署中间文件约定**：安装包、配置副本、填写中的 env、冒烟日志统一放在仓库外的工作目录：

`../ppp/deploy-artifacts/`（相对 `SE26Project-04`）

```powershell
python SE26Project-04\deploy\pack.py --out ppp\deploy-artifacts\packages --skip-frontend-build
# 或
powershell -File ppp\deploy-artifacts\rebuild-packages.ps1
```

Windows PowerShell 同样可执行：

```powershell
python deploy/pack.py
```

运行时拓扑：

```text
浏览器 → Nginx(:80/:443)
          ├─ /        前端静态资源
          ├─ /api/*   反代 → 127.0.0.1:8000（Uvicorn）
          └─ /health  反代健康检查
```

解析任务默认由 FastAPI 进程内执行，**不必**单独部署 PaperPipeline Worker。

## 2. 服务器环境要求

- Linux（推荐 Ubuntu 22.04+）
- Python **3.11+**
- Node.js **20+**（仅构建前端时需要；服务器可只部署已构建的 `dist`）
- Nginx
- 可选：PostgreSQL 14+（正式库推荐；小规模演示可用 SQLite）
- 出站网络：访问 LLM API、arXiv（订阅同步）

## 3. 后端部署

### 3.1 安装

```bash
sudo useradd -r -m -d /opt/papermate -s /bin/bash papermate || true
sudo mkdir -p /opt/papermate
sudo tar -xzf papermate-backend-*.tar.gz -C /opt/papermate
# 若解压后目录名为 backend，确保路径为 /opt/papermate/backend
cd /opt/papermate/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
# 正式库
pip install -e ".[postgres]"
# 若暂用 SQLite：pip install -e .
```

### 3.2 配置

```bash
cp .env.production.example .env   # 或从 deploy/env.backend.production.example 复制
chmod 600 .env
# 编辑：DATABASE_URL、AUTH_SECRET、CORS_ORIGINS、LLM_API_KEY、WORKER_TOKEN
```

关键变量：

| 变量 | 说明 |
|------|------|
| `PAPERMATE_ENV=prod` | 生产模式；默认关闭 `/docs` |
| `PAPERMATE_DATABASE_URL` | PostgreSQL 或 SQLite |
| `PAPERMATE_AUTH_SECRET` | 必须替换为长随机串 |
| `PAPERMATE_CORS_ORIGINS` | 公网访问来源，如 `https://域名` |
| `PAPERMATE_LLM_*` | 问答/摘要/图谱所需 |
| `PAPERMATE_ENABLE_DOCS` | 验收时可临时 `true` |
| `PAPERMATE_CRAWL_*` | 订阅定时抓取 |

### 3.3 初始化数据

```bash
cd /opt/papermate/backend
source .venv/bin/activate
python -m alembic upgrade head
# 可选导入种子论文
python -m scripts.import_seed --seed /path/to/PaperPipeline/data/seed.json
```

### 3.4 systemd

```bash
sudo cp deploy/papermate-backend.service.example /etc/systemd/system/papermate-backend.service
# 按实际路径修改 User/WorkingDirectory/ExecStart
sudo systemctl daemon-reload
sudo systemctl enable --now papermate-backend
sudo systemctl status papermate-backend
curl -f http://127.0.0.1:8000/health
```

## 4. 前端部署

构建机或 CI 上：

```bash
cd UIPrototype/frontend
cp ../../deploy/env.frontend.production.example .env.production
npm ci
npm run build
# dist/ 即前端安装内容
```

服务器上：

```bash
sudo mkdir -p /var/www/papermate
sudo tar -xzf papermate-frontend-*.tar.gz -C /tmp
sudo rsync -a /tmp/frontend/ /var/www/papermate/
sudo chown -R www-data:www-data /var/www/papermate
```

确认 `.env.production` 中：

- `VITE_USE_MOCK=false`
- `VITE_API_BASE_URL=/api`（同源反代）

## 5. Nginx

```bash
sudo cp deploy/nginx.papermate.conf.example /etc/nginx/sites-available/papermate
sudo ln -sf /etc/nginx/sites-available/papermate /etc/nginx/sites-enabled/papermate
# 修改 server_name、root、证书路径
sudo nginx -t
sudo systemctl reload nginx
```

HTTPS：用 certbot 或学校提供的证书，启用示例中的 443 server 段。

## 6. 上线冒烟清单

1. `curl -f https://域名/health` 返回 OK  
2. 打开首页，可注册/登录  
3. 工作台有推荐论文（需种子或订阅同步）  
4. 打开论文 → 触发解析 → 智能总结出现校验标记  
5. 侧栏问答返回带页码出处  
6. 设置页「立即同步 arXiv」可完成（需出站网络）  

## 7. 运维常用命令

```bash
sudo journalctl -u papermate-backend -f
sudo systemctl restart papermate-backend
# 备份 PostgreSQL
pg_dump -U papermate papermate > papermate-$(date +%F).sql
# SQLite 备份
cp /opt/papermate/backend/data/prod.db /backup/prod-$(date +%F).db
```

## 8. 安全注意

- `.env` 与 API Key **不要**提交 Git  
- 生产默认关闭 Swagger（`ENABLE_DOCS=false`）  
- 后端只监听 `127.0.0.1:8000`，不直接对公网开放  
- `AUTH_SECRET` / `WORKER_TOKEN` 使用独立随机值  

## 9. 与本地开发的差异

| 项 | 本地 | 服务器 |
|----|------|--------|
| 前端 | `npm run dev` + Vite 代理 | 静态 `dist` + Nginx |
| 后端 | 前台 uvicorn | systemd |
| 数据库 | SQLite `dev.db` | 推荐 PostgreSQL |
| Mock | 可开 | 必须关闭 |
| 文档 | `/docs` 开启 | 默认关闭 |

## 10. 可选：独立 Worker

仅在需要批量补偿解析时使用 PaperPipeline Worker，并配置相同的 `PAPERMATE_WORKER_TOKEN`。页面点击解析的主路径不依赖 Worker。
