## 三、项目主要风险及应对方案
### 风险1：功能新增过多，需求范围膨胀，工期无法按期交付
- 缓解方案：严格遵循「MVP优先、核心亮点可演示、非核心功能可降级」原则管控需求范围
- 应急方案：优先保障多Agent流水线、多模式阅读两大核心亮点；知识图谱、个性化推荐仅完成可演示轻量化版本

### 风险2：LLM生成摘要、内容校验、问答结果准确率不达标
- 缓解方案：标准化固定Prompt模板；所有AI输出内容强制绑定原文段落/论文出处；通过校验Agent自动标记模糊、不确定内容
- 应急方案：限定问答检索范围仅为本地已解析论文；提前准备人工校验标准样例用于演示

### 风险3：ArXiv爬虫抓取、PDF/HTML文档解析稳定性差
- 缓解方案：搭建论文本地缓存机制，提供手动论文导入入口；优先兼容ArXiv主流标准格式
- 应急方案：解析失败时保留论文元数据、官方摘要、原文链接及基础文献记录，保证基础展示可用

### 风险4：研究脉络、知识图谱、个性化推荐依赖大量数据，初期数据不足
- 缓解方案：基于主题、关键词、作者、实验方法、数据集、引用关系构建轻量化知识图谱；推荐模块采用「规则匹配+用户标签画像」简易方案
- 应急方案：预置标注样例数据集，采用半自动标注方式保证演示效果完整

### 风险5：100并发访问指标、接口响应时间存在性能瓶颈
- 缓解方案：将Agent大模型处理任务放入后台异步队列；检索、问答功能优先读取预生成结果，配套索引、分页、多级缓存优化
- 应急方案：单独统计LLM模型耗时，输出完整性能瓶颈分析报告与全套性能测试记录
